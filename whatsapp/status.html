Estado de la reclamación

Para un cliente retorna el estado en que está la reclamación

URL#

GET /whatsapp/clients/{id}/status

Parámetros URL#

NombreTipoComentarios
idstringIdentificador del usuario. Puede contener números y guión medio. Obligatorio

Ejemplo#

curl --location --request GET 'https://calculatuindemnizacion.es/api/whatsapp/clients/{id}/status'  \--header 'Authorization: Bearer {access_token}'
Nota

En la llamada anterior se debe sustituir {access_token} e {id} por los valores del access_token, e id real

Respuesta correcta#

La respuesta retorna los siguientes valores:

NombreTipoComentarios
statusstringCódigo de estado. Puede ser vacío o algún valor de entre los códigos de estado disponibles.

Códigos de estado#

Los códigos disponibles son RENUNCIA, DOCUMENTACION, SEGUIMIENTO_MEDICO, SERVICIOS_MEDICOS, NEGOCIANDO_CIA, PAGO_CIA, RECOBRO, SIN_CONFORMIDAD, CIERRE, VENIA, MONITORIO, FORENSE, JUDICIAL

Aviso

El código puede tener valor vacío "" si el cliente no está en ninguno de los estados disponibles

Ejemplo (devuelve un código disponible):

Status: 200 Ok
{    "status": "RENUNCIA"}

Ejemplo (devuelve vacío):

Status: 200 Ok
{    "status": ""}

Respuesta incorrecta#

Por ejemplo:

Status: 404 Not found
{    "message": "Client not found"}
